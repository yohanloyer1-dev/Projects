## Phase 1: Validate the Wedge (Weeks 1–2)
**Timeline:** Days 1–14 | **Owner:** Yohan (conversations) + Cowork (scripts)
**Environment:** Cowork (script creation, synthesis)

### 5 Target Conversations
- 3 merchants: French brands selling on ManoMano on Gorgias Basic/Pro
- 2 agency partners: Gorgias agencies with French clients

### Merchant Conversation Script (4–5 questions)

**Objective:** Confirm pain, establish WTP — without revealing the product idea.

1. "How do you currently manage customer messages that come in from ManoMano? Is it in Gorgias or a separate tool?"
2. "When a support agent opens a ManoMano ticket, what context do they wish they had that they don't? What do they have to look up manually?"
3. "How much time per week would you estimate your team spends switching between Gorgias and ManoMano's seller portal?"
4. "If there was a way to have all ManoMano messages appear as Gorgias tickets automatically — how much would that be worth to you per month? Ballpark."
5. "Who else on your team would care about this?"

### Agency Conversation Script (4–5 questions)

**Objective:** Understand client mix, confirm integration gap, gauge reseller interest.

1. "Of your Gorgias clients who sell on French marketplaces like ManoMano or Cdiscount — how do they currently handle support for those channels?"
2. "Has any client asked you to help connect ManoMano to Gorgias? What did you tell them?"
3. "If a reliable ManoMano → Gorgias integration existed, would you add it to your implementation packages? What would that be worth to clients?"
4. "What would make you comfortable recommending a third-party integration to your clients — vs. building it in-house or advising them to use a different tool?"
5. "Would you rather earn a setup fee, a monthly revenue share, or both for recommending it?"

**GitHub output:** `validation/validation-script-2026-04-30.md` (this roadmap serves as the record; extract scripts to dedicated file)
**Success metric:** ≥3 out of 5 conversations confirm: (a) ManoMano support is handled outside Gorgias, (b) WTP ≥ €50/mo (merchants) or setup fee ≥ €500 (agencies).
**Agency metric:** ≥1 agency confirms they would add it to their implementation package.

## Phase 2: Build v1 (Weeks 3–6)
**Timeline:** Days 15–42 | **Owner:** Claude Code (build) + Yohan (API access, testing)
**Environment:** Claude Code (API reading, scaffolding, testing)

### Pre-Build Gates (must be cleared before Code starts)
- [ ] Yohan confirms ManoMano Partners API access level and rate limits (T-008)
- [ ] Yohan pastes Gorgias ticket creation endpoint spec (from browser/Postman) (T-008 equivalent for Gorgias)
- [ ] Validation phase passed (≥3/5 confirmed)

### Architecture (to be confirmed by CTO Agent / Claude Code)
```
n8n schedule (every 5 min)
  → HTTP GET ManoMano Messages API (new messages since last poll)
  → For each new message:
      → Check: does Gorgias ticket exist for this ManoMano thread ID?
      → If no: POST /tickets to Gorgias (create ticket with ManoMano metadata)
      → If yes: POST /messages to existing ticket (append reply)
  → Log: ticket_id, manomano_message_id, timestamp → state store (n8n static data or simple KV)
  → On error: retry ×3, then alert via email
```

**GitHub output:** `integrations/manomano-gorgias/` (scaffolded by Claude Code)
**Success metric:** 3 test merchants with ≥100 ManoMano messages/mo successfully receiving all messages as Gorgias tickets, <30 min latency, 0 missed messages over 7-day test period.
**Agency metric:** At least 1 agency partner has seen a demo and confirmed they'd present it to a client.
