Session flow
01
Open session
Mount /Users/yohanloyer/Projects in Cowork. Global Instructions load automatically.
02
Read memory
Claude fetches CLAUDE.md, TASKS.md, DASHBOARD-TASKS.md from GitHub raw URLs. Source of truth always.
03
Route to project
Memory routing table maps the task to the right project file. Nébuleuse → nebuleuse-bijoux.md, etc.
04
Confirm in chat
✓ Read: CLAUDE.md, TASKS.md, DASHBOARD-TASKS.md (GitHub verified) — required before any work starts.
05
Do the work
Update TASKS.md immediately when status changes. Don't batch to end. Push every 1–2 hours.
06
Wrap up
Update TASKS.md + DASHBOARD-TASKS.md + session-log.md. Single git push origin main. Done.
Rules
✓ Do this
→ Always read CLAUDE.md + TASKS.md + DASHBOARD-TASKS.md at startup
→ Verify GitHub is up-to-date before working locally
→ Push every change to GitHub — every session, no exceptions
→ Update session-log.md at wrap-up, prepended (newest first)
→ Use GitHub raw URLs if no local mount available
✗ Never do this
→ Write local-only — no backup means lost work
→ Assume task status — always check TASKS.md first
→ Skip memory reads — leads to context loss
→ Leave files out of sync between local and GitHub
→ Batch TASKS.md updates to end of session
