## Common Issues & Solutions

### Issue: Dashboard won't load (404)
```
Diagnosis: GitHub Pages not deployed or URL wrong
Fix: 
  1. Verify repo is PUBLIC: github.com/yohanloyer1-dev/Projects
  2. Check GitHub Pages settings → should deploy from main branch
  3. Confirm URL is correct: yohanloyer1-dev.github.io (not github.com)
```

### Issue: TASKS.md sync fails (Auth error)
```
Diagnosis: GitHub token missing or expired
Fix:
  1. Set token in browser localStorage: 
     localStorage.setItem('yl_gist_token', 'YOUR_TOKEN')
  2. Test curl to verify token:
     curl -H "Authorization: token YOUR_TOKEN" https://api.github.com/user
  3. Try sync again
```

### Issue: Git status shows untracked files
```
Diagnosis: Test created temp files
Fix:
  1. Check which files: git status --short
  2. Remove if test artifacts: git clean -fd
  3. Or add to .gitignore if should be ignored
  4. Commit: git add .gitignore && git commit -m "Update gitignore"
```

### Issue: Performance test shows slow load time
```
Diagnosis: Network latency or browser cache issue
Fix:
  1. Repeat test 3 times, take average
  2. Clear browser cache between runs (Cmd+Shift+Delete)
  3. Check network conditions (WiFi speed, other traffic)
  4. Acceptable variance: ±20% is OK (network varies)
```

### Issue: XSS test doesn't trigger alert
```
Expected behavior! That's good.
JavaScript should NOT execute. Verify:
  1. Open browser DevTools (F12)
  2. Go to Console tab
  3. No errors should appear
  4. Type: document.querySelector('input').value
     Should show literal text, not executed code
```

## Questions During Testing?

**Common Questions:**

**Q: What if a test fails?**  
A: Document it in PHASE_4_RESULTS_TEMPLATE.md under the relevant phase. Log with severity (CRITICAL/HIGH/MEDIUM/LOW). If critical, stop and fix before continuing.

**Q: Can I skip a test?**  
A: NO. All 10 phases must complete. Skip-able items are marked in PHASE_4_TESTING_STRATEGY.md (e.g., Phase 4.5 might say "pending Phase 3" and still count as PASS).

**Q: What if GitHub is down?**  
A: Phase 4 depends on GitHub being live. Wait and re-test. Document the outage in results.

**Q: How strict are the performance targets?**  
A: Targets are guidelines. ±20% variance is acceptable (network varies). Major deviations (e.g., >6s cold load) should be investigated.

**Q: Can I parallelize testing?**  
A: Yes, 4.5, 4.9, 4.10 can run in parallel (they're independent). But run 4.1-4.7 sequentially (dependencies).

**Q: What's the minimum passing grade?**  
A: GO requires: 4.1-4.4 PASS + 4.7 PASS + 4.10 has ZERO critical findings + 4.6 all files found. Other phases can be MEDIUM/LOW findings without blocking GO.
