## 📁 Folder Organization

### 01_Quick_Reference
**Master guides and quick access documents**
- `INDEX.md` - Main navigation guide for all resources
- Central reference point for understanding the complete project

### 02_Playbooks
**Complete collection of implementation and guidance playbooks**
Contains all procedural guides for executing AI Agent implementations:
- `ai_agent_implementation_playbook.md` - Core implementation methodology
- `ai_agent_implementation_playbook_ENHANCED.md` - Enhanced version with additional details
- `ai_agent_audit_playbook.md` - Audit procedure guide
- `08_CSV_TICKET_ANALYSIS_PLAYBOOK.md` - Analyzing ticket data with CSV
- `09_GUIDANCE_AUDIT_PLAYBOOK.md` - Comprehensive audit guidance
- `10_ECONOMIC_MODEL_ROI.md` - ROI and economic modeling guide
- `11_CLIENT_REPORT_WRITING_GUIDE.md` - How to write client reports
- `12_ECOMMERCE_GUIDANCE_TEMPLATES.md` - E-commerce implementation templates

**Use this folder when:** You need step-by-step procedures, methodologies, or templates for implementation work.

### 03_Tools
**Customer discovery and audit interactive tools**
- `discovery_form_v2_professional.html` - Production discovery form
- `audit_flowchart_interactive.html` - Visual audit methodology flowchart
- `README.md` - Guide to using these tools
- `Supporting_Docs/` subfolder:
  - `CLIENT_DISCOVERY_QUESTIONS.md` - Discovery call conversation guide
  - `DISCOVERY_FORM_TESTING_CHECKLIST.md` - QA checklist
  - `FORMSPREE_SETUP_GUIDE.md` - Email integration setup
  - `POST_AUDIT_IMPLEMENTATION_CHECKLIST.md` - Post-audit execution plan

**Use this folder when:** You're preparing for customer discovery, setting up the discovery form, or explaining audit process to customers.

### 04_Session_Documentation
**Session notes, completion summaries, and audit records**
- `CURRENT_STATUS.txt` - Current project status
- `SESSION_COMPLETION_SUMMARY.md` - Latest session work completed
- `FILE_ORGANIZATION_RECOMMENDATION.md` - File organization analysis
- `NOTION_CONTENT_MAPPING.md` - Mapping of Notion content
- `NOTION_EXTRACTION_AUDIT.md` - Audit of extraction completeness
- `NOTION_EXTRACTION_VERIFICATION_RESULTS.md` - Verification results
- `ORGANIZATION_COMPLETE.txt` - Organization work summary
- `RE_EXTRACTION_COMPLETE_SUMMARY.md` - Re-extraction work summary

**Use this folder when:** You need to check what work was completed in previous sessions or understand the project status.

### Extraction_Skill
**Reusable Notion extraction methodology**
Contains the bulletproof methodology for extracting content from Notion and converting to Markdown.

**Use this folder when:** You need to extract new documents from Notion or replicate the extraction process.

### NOTION_SOURCES
**All Notion documents and knowledge bases with version control**
- `01_Active_Playbooks/` - Current production playbooks extracted from Notion
- `02_Knowledge_Bases/` - Current knowledge documents (Shopping Assistant, Support Agent)
- `03_Archives/` - Previous versions for reference
- `04_Documentation/` - VERSION_HISTORY.md and README.md
- `README.md` - Index of all extracted documents
- `VERSION_HISTORY.md` - Complete version tracking and extraction timestamps

**Use this folder when:** You need the actual extracted content, want to check version history, or verify what's been extracted from Notion.

### AI Agent + Automations Knowledge (Help Center extract)
**Help center documentation from previous extraction**
Reference material from the help center system.
