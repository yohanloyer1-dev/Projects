## 🔄 Version Control Process

### How Versions Work

When a playbook is updated in Notion:

1. **Previous version is archived** (e.g., `filename_v1.md` remains)
2. **New version is created** (e.g., `filename_v2.md`)
3. **Active version marked with `_LATEST`** (e.g., `filename_v2_LATEST.md` replaces v1_LATEST)
4. **Change is logged below** with before/after summary
5. **Both versions kept** in folder for audit trail and rollback capability

### File Naming Convention

```
{EXACT_NOTION_DOC_NAME}_v{NUMBER}[_LATEST].md

Examples:
- Success_AI_Agent_Optimization_Playbook_v1_LATEST.md (current active)
- Success_AI_Agent_Optimization_Playbook_v1.md (archived)
- Success_AI_Agent_Optimization_Playbook_v2_LATEST.md (after update)
```

## 🎯 Rollback Instructions

If you need to revert to a previous version:

1. **Identify the version to restore** from this timeline
2. **Rename the current `_LATEST` file** (e.g., remove `_LATEST` suffix)
3. **Rename the archived version** to have `_LATEST` suffix
4. **Update this VERSION_HISTORY.md** with rollback note
5. **Log the reason** for rollback in Change Log

Example:
```bash
# Rollback from v2_LATEST to v1
mv Success_AI_Agent_Optimization_Playbook_v2_LATEST.md Success_AI_Agent_Optimization_Playbook_v2.md
mv Success_AI_Agent_Optimization_Playbook_v1.md Success_AI_Agent_Optimization_Playbook_v1_LATEST.md
```
